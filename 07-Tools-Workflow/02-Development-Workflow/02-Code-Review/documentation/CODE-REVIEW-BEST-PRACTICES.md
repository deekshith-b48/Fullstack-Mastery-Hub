# Code Review Best Practices

Code review best practices improve review quality and team collaboration. This guide covers proven practices for effective code reviews.

## Best Practices Overview

Best practices include:
- **Process Practices**: Review process practices
- **Communication Practices**: Communication practices
- **Technical Practices**: Technical review practices
- **Team Practices**: Team collaboration practices
- **Continuous Improvement**: Improvement practices

## Process Practices

### Review Timing

- **Prompt Reviews**: Review within 24 hours
- **Regular Schedule**: Regular review schedule
- **Avoid Rush**: Don't rush reviews
- **Time-box**: Time-box review sessions

### Review Size

- **Small PRs**: Keep PRs under 400 lines
- **Focused Changes**: One concern per PR
- **Incremental**: Make incremental changes
- **Reviewable**: Make PRs easily reviewable

### Review Assignment

- **Appropriate Reviewers**: Assign appropriate reviewers
- **Multiple Reviewers**: Multiple reviewers for complex changes
- **Rotation**: Rotate reviewers
- **Expertise**: Match reviewer expertise

## Communication Practices

### Feedback Style

- **Constructive**: Always constructive
- **Respectful**: Respectful communication
- **Specific**: Specific feedback
- **Actionable**: Actionable suggestions
- **Positive**: Acknowledge good work

### Review Language

- **Clear**: Clear language
- **Professional**: Professional tone
- **Helpful**: Helpful suggestions
- **Explanatory**: Explain reasoning
- **Collaborative**: Collaborative approach

## Technical Practices

### Review Focus

- **Functionality**: Does it work?
- **Code Quality**: Is code well-written?
- **Tests**: Are there adequate tests?
- **Documentation**: Is it documented?
- **Standards**: Follows standards?

### Review Depth

- **Thorough**: Review thoroughly
- **Prioritize**: Focus on important issues
- **Balance**: Balance thoroughness and speed
- **Context**: Consider context
- **Impact**: Consider impact

## Common Pitfalls

### Problems to Avoid

- **Personal Attacks**: Don't attack person
- **Nitpicking**: Don't nitpick minor issues
- **Blocking Unnecessarily**: Don't block unnecessarily
- **Delayed Reviews**: Don't delay reviews
- **Vague Feedback**: Don't give vague feedback

### Solutions

- **Focus on Code**: Focus on code quality
- **Prioritize**: Prioritize important issues
- **Be Collaborative**: Work together
- **Set Expectations**: Set clear expectations
- **Provide Examples**: Provide examples
