# Task Estimation

Accurate task estimation is crucial for project planning. This guide covers estimation techniques, best practices, and common pitfalls.

## Estimation Overview

Estimation involves:
- **Effort Estimation**: Estimating effort required
- **Time Estimation**: Estimating time needed
- **Complexity Assessment**: Assessing complexity
- **Risk Consideration**: Considering risks

## Estimation Techniques

### Story Points

- **Relative Sizing**: Relative to other stories
- **Fibonacci Sequence**: 1, 2, 3, 5, 8, 13
- **Team Consensus**: Team estimates together
- **Velocity**: Track velocity over time

### Time-Based

- **Hours**: Estimate in hours
- **Days**: Estimate in days
- **Weeks**: For larger tasks
- **Historical Data**: Use historical data

### T-Shirt Sizing

- **XS, S, M, L, XL**: Size categories
- **Simple**: Simple approach
- **Quick**: Quick estimation
- **Relative**: Relative sizing

## Estimation Process

### Planning Poker

1. **Present Story**: Present user story
2. **Individual Estimates**: Team estimates individually
3. **Reveal**: Reveal estimates
4. **Discuss**: Discuss differences
5. **Re-estimate**: Re-estimate if needed

### Three-Point Estimation

- **Optimistic**: Best case scenario
- **Pessimistic**: Worst case scenario
- **Most Likely**: Most likely scenario
- **Average**: Calculate average

## Best Practices

### Estimation Guidelines

1. **Team Involvement**: Involve team
2. **Historical Data**: Use historical data
3. **Break Down**: Break down large tasks
4. **Consider Risks**: Consider risks
5. **Buffer**: Include buffer time

### Common Pitfalls

- **Over-optimism**: Being too optimistic
- **Under-estimation**: Under-estimating complexity
- **No Buffer**: No buffer time
- **Individual Estimates**: Individual vs. team
