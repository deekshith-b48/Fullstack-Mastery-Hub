# User Stories

User stories capture requirements from user perspective. This guide covers writing effective user stories and best practices.

## User Story Overview

User stories:
- **User-Focused**: Focus on user needs
- **Simple Format**: Simple, clear format
- **Conversation Starter**: Start conversations
- **Value-Driven**: Focus on value

## Story Format

### Basic Format

```
As a [user type]
I want [goal]
So that [benefit]
```

### Example

```
As a customer
I want to reset my password
So that I can regain access to my account
```

## Story Components

### User Type

- **Who**: Who is the user?
- **Role**: User's role
- **Context**: User context
- **Clear**: Clear and specific

### Goal

- **What**: What does user want?
- **Action**: Action user wants to take
- **Clear**: Clear and specific
- **Achievable**: Achievable goal

### Benefit

- **Why**: Why does user want this?
- **Value**: Value to user
- **Business Value**: Business value
- **Clear**: Clear benefit

## INVEST Criteria

### Independent

- **Standalone**: Can be developed independently
- **No Dependencies**: Minimal dependencies
- **Order Flexible**: Order doesn't matter

### Negotiable

- **Details Flexible**: Details can change
- **Discussion**: Encourages discussion
- **Collaboration**: Collaborative approach

### Valuable

- **User Value**: Value to user
- **Business Value**: Business value
- **Clear Value**: Clear value proposition

### Estimable

- **Can Estimate**: Can be estimated
- **Clear Scope**: Clear scope
- **Understandable**: Team understands

### Small

- **Manageable Size**: Can complete in sprint
- **Not Too Large**: Not epic-sized
- **Breakable**: Can break down if needed

### Testable

- **Can Test**: Can be tested
- **Clear Criteria**: Clear acceptance criteria
- **Verifiable**: Can verify completion

## Best Practices

### Writing Stories

1. **User Perspective**: Write from user perspective
2. **Clear Language**: Use clear language
3. **Specific**: Be specific
4. **Value-Focused**: Focus on value
5. **Collaborative**: Write collaboratively

### Story Refinement

1. **Regular Refinement**: Regular backlog refinement
2. **Break Down**: Break down large stories
3. **Clarify**: Clarify unclear stories
4. **Prioritize**: Prioritize stories
5. **Review**: Regular review
