# Definition of Done

Definition of Done (DoD) defines when work is complete. This guide covers creating and using DoD effectively.

## Definition of Done Overview

DoD provides:
- **Completion Criteria**: Clear completion criteria
- **Quality Standards**: Quality standards
- **Team Agreement**: Team agreement
- **Consistency**: Consistent completion

## DoD Components

### Development

- [ ] Code written
- [ ] Code reviewed
- [ ] Code merged
- [ ] Tests written
- [ ] Tests passing

### Quality

- [ ] No known bugs
- [ ] Code quality checks pass
- [ ] Performance acceptable
- [ ] Security reviewed
- [ ] Accessibility checked

### Documentation

- [ ] Code documented
- [ ] README updated
- [ ] API documentation updated
- [ ] Changelog updated
- [ ] User documentation updated

### Deployment

- [ ] Deployed to staging
- [ ] Tested in staging
- [ ] Approved for production
- [ ] Deployed to production
- [ ] Verified in production

## Creating DoD

### Process

1. **Team Discussion**: Team discusses
2. **List Criteria**: List all criteria
3. **Prioritize**: Prioritize criteria
4. **Agree**: Team agreement
5. **Document**: Document DoD

### Best Practices

1. **Team Agreement**: Full team agreement
2. **Specific**: Specific criteria
3. **Testable**: Testable criteria
4. **Realistic**: Realistic criteria
5. **Evolving**: Evolve over time

## Using DoD

### Application

- **Every Story**: Apply to every story
- **Consistently**: Apply consistently
- **Review**: Review against DoD
- **Enforce**: Enforce DoD
- **Update**: Update as needed

### Benefits

- **Quality**: Ensures quality
- **Consistency**: Consistent completion
- **Clarity**: Clear expectations
- **Accountability**: Accountability
