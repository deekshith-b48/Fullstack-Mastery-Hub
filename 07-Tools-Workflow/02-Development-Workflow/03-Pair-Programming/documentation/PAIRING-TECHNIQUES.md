# Pairing Techniques

Various pairing techniques adapt pair programming to different situations. This guide covers different pairing approaches and when to use them.

## Technique Types

### Ping-Pong Pairing

**Approach:**
- Driver writes failing test
- Navigator makes it pass
- Navigator writes next failing test
- Driver makes it pass
- Repeat

**Best For:**
- Test-driven development
- Learning TDD
- Ensuring test coverage

### Strong-Style Pairing

**Approach:**
- Navigator gives instruction
- Driver implements exactly
- No driver initiative
- Navigator thinks ahead

**Best For:**
- Knowledge transfer
- Mentoring
- Complex problems

### Driver-Navigator

**Approach:**
- Driver types code
- Navigator reviews and guides
- Regular role switching
- Collaborative approach

**Best For:**
- General development
- Most situations
- Balanced collaboration

## Specialized Techniques

### Mob Programming

**Approach:**
- Entire team pairs together
- One driver, rest navigate
- Rotate driver frequently
- Collaborative problem solving

**Best For:**
- Complex problems
- Team alignment
- Knowledge sharing

### Expert-Novice Pairing

**Approach:**
- Expert guides
- Novice learns
- Knowledge transfer focus
- Teaching opportunity

**Best For:**
- Mentoring
- Onboarding
- Skill development

### Distributed Pairing

**Approach:**
- Remote pairing
- Digital tools
- Video communication
- Screen sharing

**Best For:**
- Remote teams
- Distributed teams
- Flexible collaboration
