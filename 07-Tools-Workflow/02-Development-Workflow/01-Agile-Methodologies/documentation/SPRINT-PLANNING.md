# Sprint Planning

Sprint Planning is a Scrum event where the team plans work for the upcoming sprint. This guide covers sprint planning process, best practices, and techniques.

## Sprint Planning Overview

Sprint Planning purpose:
- **Plan Sprint**: Plan work for next sprint
- **Set Goal**: Define sprint goal
- **Select Items**: Choose backlog items
- **Create Plan**: Create sprint backlog
- **Commit**: Team commits to sprint goal

## Sprint Planning Process

### Two Parts

**Part 1: What to Build**
- Review product backlog
- Select sprint items
- Define sprint goal
- Duration: ~2 hours

**Part 2: How to Build**
- Break down items
- Create tasks
- Estimate effort
- Duration: ~2 hours

### Total Duration

- **2-week Sprint**: 4 hours
- **1-week Sprint**: 2 hours
- **Time-boxed**: Strict time limit

## Participants

### Required Attendees

- **Product Owner**: Presents backlog
- **Scrum Master**: Facilitates meeting
- **Development Team**: Plans work

### Preparation

- **Product Owner**: Prepare backlog
- **Team**: Review backlog items
- **Environment**: Set up meeting space
- **Tools**: Prepare planning tools

## Planning Activities

### Part 1: What

**Activities:**
1. **Review Backlog**: Review product backlog
2. **Discuss Items**: Discuss items with team
3. **Select Items**: Select items for sprint
4. **Define Goal**: Create sprint goal
5. **Confirm Selection**: Confirm selected items

### Part 2: How

**Activities:**
1. **Break Down**: Break down items into tasks
2. **Estimate**: Estimate task effort
3. **Create Tasks**: Create task list
4. **Assign**: Assign tasks (optional)
5. **Verify Capacity**: Verify team capacity

## Sprint Goal

### Definition

Sprint goal is:
- **Objective**: Clear objective for sprint
- **Unifying**: Unifies team work
- **Flexible**: Allows flexibility in implementation
- **Measurable**: Can measure success

### Example Goals

- "Implement user authentication"
- "Improve mobile app performance"
- "Add payment processing"
- "Fix critical bugs"

## Capacity Planning

### Velocity

- **Historical Velocity**: Use past sprint velocity
- **Team Capacity**: Consider team availability
- **Buffer**: Include buffer for unknowns
- **Realistic**: Be realistic about capacity

### Factors

- **Team Size**: Number of team members
- **Availability**: Days available
- **Holidays**: Account for holidays
- **Other Work**: Other commitments

## Best Practices

### Planning Tips

1. **Prepare**: Product Owner prepares backlog
2. **Time-box**: Stick to time limits
3. **Collaborate**: Team collaborates on plan
4. **Realistic**: Be realistic about capacity
5. **Focus**: Focus on sprint goal

### Common Pitfalls

- **Over-commitment**: Committing too much
- **Under-commitment**: Committing too little
- **Vague Goal**: Unclear sprint goal
- **Skipping Breakdown**: Not breaking down items

## Tools

### Planning Tools

- **Physical Board**: Whiteboard with sticky notes
- **Digital Tools**: Jira, Azure DevOps, Trello
- **Hybrid**: Combination of physical and digital
