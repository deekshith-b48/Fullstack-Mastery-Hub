# Retrospectives

Sprint Retrospective is a Scrum event where the team reflects and improves. This guide covers retrospective formats, techniques, and best practices.

## Retrospective Overview

Retrospective purpose:
- **Reflect**: Reflect on sprint
- **Identify Improvements**: Find improvement opportunities
- **Create Action Items**: Create actionable improvements
- **Team Growth**: Help team grow
- **Continuous Improvement**: Drive continuous improvement

## Retrospective Format

### Typical Structure

1. **Set the Stage**: Create safe environment
2. **Gather Data**: Collect feedback
3. **Generate Insights**: Identify patterns
4. **Decide What to Do**: Create action items
5. **Close**: Wrap up retrospective

### Duration

- **2-week Sprint**: 1-2 hours
- **1-week Sprint**: 30-60 minutes
- **Time-boxed**: Stick to time limit

## Retrospective Techniques

### Start, Stop, Continue

- **Start**: What should we start doing?
- **Stop**: What should we stop doing?
- **Continue**: What should we continue doing?

### Mad, Sad, Glad

- **Mad**: What made us mad?
- **Sad**: What made us sad?
- **Glad**: What made us glad?

### 4Ls

- **Liked**: What did we like?
- **Learned**: What did we learn?
- **Lacked**: What did we lack?
- **Longed For**: What did we long for?

### Sailboat

- **Wind**: What helped us?
- **Anchors**: What slowed us down?
- **Rocks**: What are risks?
- **Destination**: What is our goal?

## Facilitation

### Scrum Master Role

- **Facilitate**: Facilitate meeting
- **Create Safe Space**: Ensure psychological safety
- **Guide Discussion**: Guide discussion
- **Document**: Document action items

### Team Participation

- **Everyone Participates**: All team members
- **Open Communication**: Open, honest communication
- **Respect**: Respect different opinions
- **Focus**: Focus on improvement

## Action Items

### Creating Action Items

- **Specific**: Be specific
- **Actionable**: Actionable items
- **Owned**: Assign owners
- **Trackable**: Track progress

### Follow-up

- **Review**: Review in next retrospective
- **Track**: Track action item progress
- **Celebrate**: Celebrate improvements
- **Adjust**: Adjust if needed

## Best Practices

### Retrospective Tips

1. **Safe Environment**: Create safe environment
2. **Focus on Process**: Focus on process, not people
3. **Action Items**: Create actionable items
4. **Follow-up**: Follow up on action items
5. **Variety**: Vary techniques

### Common Pitfalls

- **Complaint Session**: Don't make it a complaint session
- **No Action Items**: Always create action items
- **Same Technique**: Vary techniques
- **No Follow-up**: Follow up on action items
