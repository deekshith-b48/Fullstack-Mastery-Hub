# Agile Metrics

Agile metrics help teams measure progress, performance, and improvement. This guide covers key Agile metrics, how to measure them, and best practices.

## Metrics Overview

Agile metrics provide:
- **Visibility**: Make progress visible
- **Insights**: Provide insights into performance
- **Improvement**: Drive continuous improvement
- **Decision Making**: Support decision making
- **Accountability**: Create accountability

## Key Metrics

### Velocity

**Definition**: Work completed per sprint

**Measurement:**
- Story points completed
- Features delivered
- User stories completed

**Usage:**
- Predict future sprints
- Plan capacity
- Track trends

### Burn-down Chart

**Definition**: Visual representation of work remaining

**Shows:**
- Work remaining over time
- Progress toward sprint goal
- Trend analysis

**Usage:**
- Track sprint progress
- Identify issues early
- Adjust plan if needed

### Burn-up Chart

**Definition**: Visual representation of work completed

**Shows:**
- Work completed over time
- Progress toward goal
- Scope changes

**Usage:**
- Track progress
- Show scope changes
- Demonstrate value

### Lead Time

**Definition**: Time from request to delivery

**Measurement:**
- Start to finish time
- Request to release time

**Usage:**
- Measure delivery speed
- Identify bottlenecks
- Improve flow

### Cycle Time

**Definition**: Time work is actively being worked on

**Measurement:**
- Time in active work
- Development time

**Usage:**
- Measure efficiency
- Identify bottlenecks
- Optimize flow

### Throughput

**Definition**: Work completed per time period

**Measurement:**
- Items completed per sprint
- Features delivered per month

**Usage:**
- Measure productivity
- Track trends
- Predict delivery

## Quality Metrics

### Defect Rate

**Definition**: Number of defects per feature

**Measurement:**
- Bugs found per feature
- Defects per story point

**Usage:**
- Measure quality
- Identify quality issues
- Improve quality

### Test Coverage

**Definition**: Percentage of code covered by tests

**Measurement:**
- Code coverage percentage
- Test coverage metrics

**Usage:**
- Measure test quality
- Identify gaps
- Improve coverage

## Team Metrics

### Team Satisfaction

**Definition**: Team member satisfaction

**Measurement:**
- Surveys
- Retrospective feedback

**Usage:**
- Measure team health
- Identify issues
- Improve team culture

### Sprint Goal Achievement

**Definition**: Percentage of sprint goals achieved

**Measurement:**
- Goals achieved vs. planned
- Success rate

**Usage:**
- Measure planning accuracy
- Improve planning
- Track consistency

## Best Practices

### Metrics Selection

1. **Relevant**: Choose relevant metrics
2. **Actionable**: Metrics should drive action
3. **Balanced**: Balance different metrics
4. **Simple**: Keep metrics simple
5. **Visible**: Make metrics visible

### Using Metrics

1. **Trend Analysis**: Look at trends, not single points
2. **Context**: Consider context
3. **Team Input**: Get team input
4. **Action**: Take action on insights
5. **Review**: Regularly review metrics

## Common Pitfalls

### Problems to Avoid

- **Vanity Metrics**: Avoid vanity metrics
- **Over-measurement**: Don't measure everything
- **Gaming**: Prevent metric gaming
- **Punishment**: Don't use metrics to punish

### Solutions

- **Focus Value**: Focus on value metrics
- **Team Ownership**: Team owns metrics
- **Improvement Focus**: Focus on improvement
- **Transparency**: Keep metrics transparent
