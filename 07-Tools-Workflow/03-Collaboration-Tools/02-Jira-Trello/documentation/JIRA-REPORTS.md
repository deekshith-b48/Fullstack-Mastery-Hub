# Jira Reports

Jira reports provide insights into project progress and team performance. This guide covers report types, usage, and best practices for data-driven project management.

## Report Overview

Reports provide:
- **Progress Tracking**: Track project progress
- **Team Metrics**: Team performance metrics
- **Sprint Analytics**: Sprint analysis
- **Burndown Charts**: Burndown visualization
- **Velocity Tracking**: Velocity tracking

## Report Types

### Sprint Reports

- **Burndown Chart**: Sprint burndown
- **Sprint Report**: Sprint summary
- **Velocity Chart**: Velocity tracking
- **Sprint Health**: Sprint health metrics

### Agile Reports

- **Epic Report**: Epic progress
- **Version Report**: Version progress
- **Cumulative Flow**: Cumulative flow diagram
- **Control Chart**: Cycle time analysis

### Issue Reports

- **Created vs Resolved**: Issue trends
- **Time in Status**: Time analysis
- **Assignee Report**: Assignee workload
- **Resolution Time**: Resolution time analysis

## Using Reports

### Accessing Reports

1. **Project**: Go to project
2. **Reports**: Select Reports
3. **Choose Report**: Select report type
4. **Configure**: Configure parameters
5. **View**: View report

### Report Configuration

- **Date Range**: Set date range
- **Filters**: Apply filters
- **Parameters**: Configure parameters
- **Export**: Export reports

## Best Practices

### Report Usage

1. **Regular Review**: Review reports regularly
2. **Team Sharing**: Share with team
3. **Action Items**: Act on insights
4. **Trend Analysis**: Analyze trends
5. **Improvement**: Use for improvement
